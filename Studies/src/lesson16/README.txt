Пакет (package)
------------------

Пакет представляет собой набор(группу) классов, интерфейсов
и других пакетов.

В Java используем пакеты для организации классов и
интерфейсов.

Типы пакетов в Java:
- встроенные пакеты (в JDK);
- пользовательские пакеты (создаются разработчиком).

Преимущества использования пакета на Java:
1) Возможность повторного использования: при разработке проекта
возникает повторяемость кода. Используя пакеты, можно создать, например,
служебный класс в служебном пакете. Когда нужно выполнить ту же
задачу, просто импортируйте этот пакет и используйте класс.
2) Оптимальная организация кода: в больших проектах, где есть, например,
несколько сотен классов, всегда требуется сгруппировать
похожие типы классов в значимое имя пакета, чтобы можно было лучше
организовать свой проект, и когда нужно что-то, можно быстро
найти его и использовать, что повышает эффективность/производительность.
3) Избежание конфликта имен: можно определить два класса с одинаковыми
именами в разных пакетах, чтобы избежать конфликта имен.

Импорт
------------
В Java есть оператор import, позволяющий импортировать весь пакет
или использовать только определенные классы и интерфейсы, определенные
в пакете.
Общая форма оператора import:
1) импортировать только определенный класс
import package.name.ClassName;
2) импортировать весь пакет
import package.name.*


Статический импорт Java
--------------------------
Функция статического импорта обеспечивает прямой доступ к любому статическому
члену класса. Нет необходимости уточнять его по имени класса.
Преимущество статического импорта:
Если часто присутствует доступ к любому статическому члену класса, требуется
меньше кода.
Недостаток статического импорта:
Если чрезмерно используете функцию статического импорта, это сделает
программу нечитаемой и трудной для обслуживания.
